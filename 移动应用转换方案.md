# 歌词旋律配对工具 - 移动应用转换方案

## 📱 转换方案概述

将微信小程序转换为原生移动应用（iOS/Android）有以下几种方案：

### 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **1. UniApp跨平台** | 一套代码多端运行，Vue语法，生态完善 | 性能略低于原生，包体积较大 | 快速上线多平台 |
| **2. React Native** | 性能接近原生，热更新，组件丰富 | 需要重写逻辑，学习曲线 | 高性能要求 |
| **3. Flutter** | 性能优秀，UI美观，跨平台 | Dart语言，生态较小 | 对性能要求高 |
| **4. 原生开发** | 性能最优，功能完整 | 成本高，维护复杂 | 企业级应用 |
| **5. Taro框架** | 支持多端，React语法，小程序友好 | 性能中等 | 有小程序基础 |

## 🎯 推荐方案：UniApp跨平台开发

基于您已有的微信小程序基础，**UniApp是最佳选择**：
- ✅ Vue语法，学习成本低
- ✅ 支持微信、H5、Android、iOS多端
- ✅ 丰富的组件库和插件
- ✅ 可以复用大部分小程序逻辑

## 📋 转换步骤

### 第一阶段：项目初始化

1. **安装UniApp开发环境**
   ```bash
   # 安装HBuilderX IDE
   # 或者使用Vue CLI
   npm install -g @vue/cli
   vue create -p dcloudio/uni-preset-vue lyrics-app
   ```

2. **项目结构迁移**
   ```
   lyrics-app/
   ├── pages/          # 页面（从小程序迁移）
   ├── components/     # 组件
   ├── static/         # 静态资源
   ├── uni_modules/    # UniApp插件
   ├── App.vue         # 应用入口
   ├── main.js         # 主入口文件
   └── manifest.json   # 应用配置
   ```

### 第二阶段：核心功能转换

#### 1. 数据存储转换

**微信小程序云数据库 → UniApp云开发**
```javascript
// 小程序云开发
wx.cloud.collection('works').get()

// UniApp云开发（使用uniCloud）
uniCloud.callFunction({
  name: 'getWorks',
  data: {}
})
```

**本地存储转换**
```javascript
// 小程序
wx.setStorageSync('key', value)
wx.getStorageSync('key')

// UniApp
uni.setStorageSync('key', value)
uni.getStorageSync('key')
```

#### 2. 音频功能转换

```javascript
// 小程序录音
wx.startRecord()

// UniApp录音
uni.startRecord()

// 播放音频
// 小程序
wx.createInnerAudioContext()

// UniApp
uni.createInnerAudioContext()
```

#### 3. UI组件转换

| 小程序组件 | UniApp组件 |
|-----------|------------|
| `<view>` | `<view>` |
| `<text>` | `<text>` |
| `<button>` | `<button>` |
| `<image>` | `<image>` |
| `<scroll-view>` | `<scroll-view>` |
| `<swiper>` | `<swiper>` |

#### 4. API转换示例

```javascript
// 网络请求
// 小程序
wx.request({ url: '...' })

// UniApp
uni.request({ url: '...' })

// 导航
// 小程序
wx.navigateTo({ url: '...' })

// UniApp
uni.navigateTo({ url: '...' })

// 系统信息
// 小程序
wx.getSystemInfoSync()

// UniApp
uni.getSystemInfoSync()
```

### 第三阶段：云服务配置

#### 1. uniCloud配置

```javascript
// manifest.json
{
  "uniCloud": true,
  "cloudfunctionRoot": "cloudfunctions/"
}

// 云函数示例
'use strict';
exports.main = async (event, context) => {
  const db = uniCloud.database()
  const res = await db.collection('works').get()
  return res
}
```

#### 2. 数据库迁移

```javascript
// 云数据库集合定义
const db = uniCloud.database()

// 创建集合（在云开发控制台操作）
// - works: 作品数据
// - songs: 歌曲数据
// - lyrics: 歌词数据
// - pairings: 配对记录
```

### 第四阶段：打包发布

#### 1. Android打包

```bash
# 在HBuilderX中
1. 发行 -> 原生App-云打包
2. 选择Android平台
3. 配置签名证书
4. 生成APK文件
```

#### 2. iOS打包

```bash
# 在HBuilderX中
1. 发行 -> 原生App-云打包
2. 选择iOS平台
3. 配置证书和描述文件
4. 生成IPA文件
```

#### 3. 应用上架

**Android：**
1. 注册Google Play开发者账号（$25）
2. 创建应用
3. 上传APK
4. 填写应用信息
5. 提交审核

**iOS：**
1. 注册Apple Developer账号（$99/年）
2. 创建App ID
3. 配置证书和描述文件
4. 上传到App Store Connect
5. 提交审核

## 🔧 技术栈转换对比

### 原生技术栈
- **小程序**: 微信小程序原生框架
- **数据库**: 微信云开发数据库
- **存储**: 微信云存储
- **语言**: JavaScript + WXML + WXSS

### 转换后技术栈
- **框架**: UniApp + Vue 3
- **数据库**: uniCloud (阿里云/腾讯云)
- **存储**: uniCloud存储
- **语言**: JavaScript/TypeScript + Vue SFC

## 📦 核心功能实现

### 1. 歌词管理页面

```vue
<template>
  <view class="lyrics-container">
    <view v-for="(lyric, index) in lyrics" :key="index">
      <text>{{ lyric.content }}</text>
      <text>类型: {{ lyric.type }}</text>
    </view>
  </view>
</template>

<script>
export default {
  data() {
    return {
      lyrics: []
    }
  },
  methods: {
    async loadLyrics() {
      const res = await uniCloud.callFunction({
        name: 'getLyrics',
        data: { workId: this.workId }
      })
      this.lyrics = res.result.data
    }
  }
}
</script>
```

### 2. 音频录制功能

```javascript
// 开始录音
const recorderManager = uni.getRecorderManager()

recorderManager.start({
  format: 'mp3',
  duration: 300000
})

// 停止录音
recorderManager.stop()

recorderManager.onStop((res) => {
  console.log('录音文件:', res.tempFilePath)
  // 上传到云存储
  this.uploadAudio(res.tempFilePath)
})
```

### 3. 音频播放器

```javascript
const innerAudioContext = uni.createInnerAudioContext()

innerAudioContext.src = audioUrl
innerAudioContext.play()

innerAudioContext.onTimeUpdate(() => {
  // 更新歌词显示
  this.updateLyrics(innerAudioContext.currentTime)
})
```

```javascript
// 主题切换
export default {

      this.$store.commit('setTheme', theme)
    }
  }
}
```
