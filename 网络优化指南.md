## 优化方案

### 方案1：运行优化脚本（推荐）

```bash
sudo ~/CodeBuddy/Claw/网络优化脚本.sh
```

**优化内容**：
- ✓ 设置阿里云 DNS (223.5.5.5, 223.6.6.6)
- ✓ 增加 TCP 缓冲区到 131072 字节
- ✓ 增加 TCP 连接队列到 1024
- ✓ 增加 TCP 最大连接数到 2048
- ✓ 启用 TCP 窗口缩放

### 方案2：手动设置 DNS

```bash
# 设置阿里云 DNS（推荐）
networksetup -setdnsservers Wi-Fi 223.5.5.5 223.6.6.6

# 或设置腾讯 DNS
networksetup -setdnsservers Wi-Fi 119.29.29.29

# 或设置百度 DNS
networksetup -setdnsservers Wi-Fi 180.76.76.76
```

### 方案3：清除 DNS 缓存

```bash
# 清除 DNS 缓存
sudo dscacheutil -flushcache
sudo killall -HUP mDNSResponder
```

### 方案4：重置网络配置

```bash
# 重置 DNS 设置
networksetup -setdnsservers Wi-Fi empty

# 重新启动 Wi-Fi
networksetup -setairportpower en0 off
sleep 2
networksetup -setairportpower en0 on
```

## GitHub 访问优化

### 方法1：配置 Git 使用代理

```bash
# 设置代理（需要你有代理服务）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy https://127.0.0.1:7890

# 取消代理
git config --global --unset http.proxy
git config --global --unset https.proxy
```

### 方法2：配置 Git 使用镜像

```bash
# 配置 GitHub 镜像
git config --global url."https://github.com.cnpmjs.org/".insteadOf "https://github.com/"

# 或使用 jsdelivr CDN
git config --global url."https://cdn.jsdelivr.net/gh/".insteadOf "https://raw.githubusercontent.com/"
```

### 方法3：修改 hosts 文件

```bash
# 编辑 hosts 文件
sudo nano /etc/hosts

# 添加以下内容（根据最新的 GitHub IP）
# 示例（需要查询最新 IP）：
# 140.82.112.4 github.com
# 140.82.113.4 github.global.ssl.fastly.net
```

## 常见问题

### Q: 优化后 GitHub 仍然无法连接？
A: 可能需要配置代理或使用镜像源，参考上面的 GitHub 访问优化方法。

### Q: 网络变得不稳定？
A: 可以恢复默认设置：
```bash
networksetup -setdnsservers Wi-Fi empty
```

### Q: 如何查看当前 DNS 设置？
```bash
networksetup -getdnsservers Wi-Fi
```

### Q: 如何查看当前网络配置？
```bash
# 查看网络接口
scutil --nwi

# 查看网络状态
ifconfig

# 查看路由表
netstat -rn
```
